# Учебнаяа практика
Десктопное приложение для управления сетевыми хранилищами


## 1. Выбор темы проекта и обоснование модели проектирования

В качестве темы проекта было дано задание создание десктопного приложения для управления сетевыми хранилищами. Данная тема с работой с сетевыми дисками и файловыми серверами является распространённой задачей, особенно в сфере IT. При этом существующие решения либо работают через браузер, либо имеют ограниченный функционал, поэтому разработка отдельного десктопного приложения выглядит актуальной и полезной.

В рамках проекта планируется реализовать программу, которая позволит подключаться к сетевым хранилищам, просматривать файлы и выполнять основные операции с ними. Это делает проект практико-ориентированным и полезным для дальнейшего применения.

При выборе модели разработки рассматривались два подхода: Waterfall и Agile. Модель Waterfall предполагает строгую последовательность этапов, где каждый следующий начинается только после завершения предыдущего. Такой подход удобен, если все требования заранее известны и не изменяются.

Однако в данном проекте требования могут уточняться по мере разработки, а функциональность может постепенно расширяться. Поэтому более подходящей является модель Agile, которая позволяет работать итерациями, постепенно добавляя новые возможности и внося изменения при необходимости.

Таким образом, для разработки данного проекта была выбрана методология Agile, так как она лучше подходит для гибкой и поэтапной разработки приложения.
